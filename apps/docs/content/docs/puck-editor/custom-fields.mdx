Puck supports custom field types beyond the built-in text, textarea, number, select, and radio controls. Custom fields use `type: "custom"` with a `render` function that receives `value`, `onChange`, and `readOnly` props.

## Media Picker Field

The media picker allows editors to upload images to Shopify or browse the existing media library. It uses Shopify's staged upload API for secure file handling, while the editor talks to the backend through tRPC procedures.

**Location:** `src/app/cms/_components/editor/media-picker/`

### Usage

Use the `mediaPickerFieldConfig()` factory function in any block config:

```tsx
import { mediaPickerFieldConfig } from "@cms/_components/editor/media-picker";

export const myBlockConfig: ComponentConfig<MyBlockProps> = {
  fields: {
    heroImage: mediaPickerFieldConfig("Hero Image"),
  },
  // ...
};
```

It also works as a sub-field inside array fields:

```tsx
categories: {
  type: "array",
  arrayFields: {
    name: { type: "text", label: "Name" },
    image: mediaPickerFieldConfig("Image"),
  },
},
```

### Features

- **Drag-and-drop upload** — drop an image onto the field to upload
- **Upload button** — click to select a file from disk
- **Browse Library** — modal dialog to browse and search existing Shopify media
- **Image preview** — shows a thumbnail of the selected image
- **Remove** — clear the selected image

### Architecture

The media picker consists of:

| File                            | Purpose                                                                                      |
| ------------------------------- | -------------------------------------------------------------------------------------------- |
| `media-picker-field-config.tsx` | Factory function returning Puck custom field config                                          |
| `media-picker-field.tsx`        | Main UI component with upload zone and controls                                              |
| `media-library-modal.tsx`       | Dialog for browsing existing Shopify media                                                   |
| `use-media-upload.ts`           | React Query mutation hook that calls tRPC media procedures and performs the direct file POST |
| `use-media-library.ts`          | Infinite query hook backed by `trpc.media.list`                                              |
| `src/trpc/routers/media.ts`     | Server-side staged upload, file registration, and media browsing procedures                  |

### Upload Flow

File uploads go through Shopify's 3-step staged upload process:

1. **Get staged URL** — `trpcClient.media.stagedUpload.mutate()` calls `stagedUploadsCreate`
2. **Upload to staged URL** — browser POSTs the file directly to Shopify's storage
3. **Register the file** — `trpcClient.media.create.mutate()` calls `fileCreate` to finalize

After upload, the media library query cache is invalidated with `trpc.media.list.queryKey()` so the new image appears in the browse modal.

See [Media Uploads](/docs/shopify-integration/media-uploads) for the server-side procedure details.

## Collection Picker Field

The collection picker lets authors search Shopify collections instead of typing collection handles manually. It is used for the root-level `collectionHandle` field on collection templates and for block fields such as `ProductCategory.categories[].href`.

**Location:** `src/app/cms/_components/editor/collection-picker/`

### Usage

Use the `collectionPickerFieldConfig()` factory anywhere a field should store a Shopify collection handle:

```tsx
import { collectionPickerFieldConfig } from "@cms/_components/editor/collection-picker";

export const puckConfig: Config<Props, CmsRootProps> = {
  root: {
    fields: {
      collectionHandle: collectionPickerFieldConfig("Collection Handle"),
    },
  },
};
```

It also works inside array fields:

```tsx
categories: {
  type: "array",
  arrayFields: {
    name: { type: "text", label: "Name" },
    image: mediaPickerFieldConfig("Image"),
    href: collectionPickerFieldConfig("Collection"),
  },
},
```

### Features

- **Debounced search** — waits briefly before querying Shopify as the author types
- **Collection metadata** — shows collection title, handle, and image when available
- **Single-click selection** — stores the selected collection handle in Puck data
- **Clear action** — removes the current selection without editing raw JSON

### Architecture

| File                                                | Purpose                                                        |
| --------------------------------------------------- | -------------------------------------------------------------- |
| `collection-picker-field-config.tsx`                | Factory function returning Puck custom field config            |
| `collection-picker-field.tsx`                       | Search input, selected-value chip, and results dropdown        |
| `use-collection-search.ts`                          | React Query hook backed by `trpc.cms.searchCollections`        |
| `src/trpc/routers/cms.ts`                           | tRPC procedures for collection lookup and collection search    |
| `src/lib/shopify/queries/collections.ts`            | Storefront search DTO mapping and `searchCollections()` helper |
| `src/lib/shopify/queries/collections.storefront.ts` | Storefront GraphQL document for collection search              |

The picker stores collection handles, not full URLs. Storefront components can then convert those handles into route paths such as `/collections/{handle}`.

## Built-in Field Overrides

The editor also restyles several built-in Puck field types in `src/app/cms/_components/editor/editor-field-types.tsx`. That now includes `radio`, which is used for root-level visibility controls such as **Site Header** and **Site Footer**.

These are still standard Puck field types — the project overrides their rendering so they match the CMS UI without redefining them as custom fields.

## Creating a Custom Field

To create a new custom field, follow the factory function pattern:

```tsx
// color-picker-field-config.tsx
import { ColorPickerField } from "./color-picker-field";

export function colorPickerFieldConfig(label = "Color") {
  return {
    type: "custom" as const,
    label,
    render: ({
      value,
      onChange,
      readOnly,
    }: {
      value: string;
      onChange: (value: string) => void;
      readOnly?: boolean;
    }) => (
      <ColorPickerField value={value} onChange={onChange} readOnly={readOnly} />
    ),
  };
}
```

The factory function pattern lets you reuse the field across blocks and as sub-fields in array fields, with customizable labels and options per usage.
