The CMS uses two Shopify APIs:

- **Admin API** — For reading and writing CMS page data (Metaobjects)
- **Storefront API** — For public-facing reads of published pages

Both use GraphQL with auto-generated TypeScript types.

## API Clients

Two thin `fetch` wrappers in `src/lib/shopify/`:

| Client     | File                   | Header                              | Use Case              |
| ---------- | ---------------------- | ----------------------------------- | --------------------- |
| Admin      | `admin-client.ts`      | `X-Shopify-Access-Token`            | CMS CRUD, dashboard   |
| Storefront | `storefront-client.ts` | `X-Shopify-Storefront-Access-Token` | Public page rendering |

Both share the same error types (`ShopifyApiError`, `ShopifyNetworkError`) defined in `src/lib/shopify/types.ts`.

## Data Storage

CMS page content is stored in Shopify Metaobjects (type: `cms_page`). Each page is a metaobject instance with fields for title, slug, Puck JSON data, status, and timestamp.

See [Metaobject Schema](/docs/shopify-integration/metaobject-schema) for the full field definition.

## Storefront Commerce Pages

Storefront collection and product routes now share the same Shopify product DTOs, route helpers, and CMS-template pattern.

- **Collections** — `/collections/[handle]` can render a published `pageType: "collection"` CMS template, or fall back to the built-in collection grid. Collection cards use Shopify gallery images from `images(first: 10)` and now deep-link into internal product pages.
- **Products** — `/products/[handle]` can render a published `pageType: "product"` CMS template, or fall back to the built-in `ProductDetail` UI with a gallery, option selectors, price/compare-at state, and stock-aware actions.
- **Shared routing** — `src/lib/routes.ts` centralizes `getCollectionPath()` and `getProductPath()` so storefront components and CMS blocks stop hardcoding URL strings.

See [Storefront Collections](/docs/shopify-integration/storefront-collections) and [Storefront Products](/docs/shopify-integration/storefront-products) for the full flow.

## Type Safety

All GraphQL operations are type-checked at compile time using `@shopify/api-codegen-preset`. Queries are defined in `*.admin.ts` and `*.storefront.ts` files with `#graphql` tags, and codegen produces typed response/variable interfaces.

See [GraphQL Codegen](/docs/shopify-integration/graphql-codegen) for setup details.
